# 脑卒中预后 Label 抽取 Prompts

项目中的建模 CSV 是派生表，原始数据来源应追溯到：

- MIMIC-III Waveform Database Matched Subset v1.0：`https://physionet.org/content/mimic3wdb-matched/1.0/`
- MC-MED v1.0.0：`https://physionet.org/content/mc-med/1.0.0/`

注意：MIMIC matched subset 本身主要提供与 MIMIC-III 临床数据库匹配的波形和数值记录。若要抽取预后 label，需要结合匹配到的临床表、出院信息、死亡信息或人工整理的病例级轨迹文本。

## Label 定义

| 建模字段 | 标签 | 含义 |
| :-: | :--: | :-- |
| `LABEL=0` | favorable / Improved | 预后良好：病情改善、症状缓解、出院回家、居家健康服务等。 |
| `LABEL=1` | unfavorable / Worsened_or_Died / Worsened_or_Severe | 预后不良：死亡、病情恶化、严重残疾、临终关怀、转入康复/长期照护/熟练护理机构、急性医院间转院等。 |
| `LABEL=null` | uncertain | 证据不足、结局冲突、只有诊断无结局、无法判断改善或恶化。 |

## 抽取 Prompt

### System Prompt

```text
你是一名临床数据标注专家，负责从脑卒中相关就诊记录中抽取二分类预后 label。

你必须只依据输入中提供的结构化字段和文本证据判断，不得根据年龄、诊断名称、生命体征、实验室异常或模型风险分数推测结局。

输出必须是合法 JSON，不要输出 Markdown、解释性段落或多余文本。

标签定义：
- favorable：明确提示预后良好、病情改善、症状缓解、神经功能好转、出院回家、home、home health care、home with services。
- unfavorable：明确提示死亡、expired、died、hospice、病情恶化、worsened、deteriorated、severe disability、转入 skilled nursing facility、inpatient rehabilitation、long-term care、nursing home、acute-care transfer、transfer to another hospital。
- uncertain：没有明确结局证据、字段缺失、仅有卒中诊断或治疗过程、证据互相冲突。

判定规则：
1. 死亡、临终关怀、明确恶化或严重残疾一律优先判为 unfavorable。
2. 明确回家或居家健康服务，且没有死亡、恶化、严重残疾或高级别照护证据，判为 favorable。
3. “stable/稳定”不能单独判为 favorable，除非同时出现回家、改善或低级别照护证据。
4. “admitted/住院/ICU”只能说明去向或病情严重程度，若没有最终预后或出院去向证据，应输出 uncertain。
5. 同一条记录中 favorable 与 unfavorable 证据冲突时，输出 uncertain，并在 conflict 字段说明冲突。
6. evidence 必须摘录输入中的原始字段值或原文短语。
```

### User Prompt 模板

```text
请为以下脑卒中相关记录抽取预后 label。

输入记录：
{{record_json_or_text}}

请按以下 JSON schema 输出：
{
  "record_id": "输入中的记录编号；若没有则填 null",
  "label": "favorable | unfavorable | uncertain",
  "label_code": 0,
  "binary_prognosis": "Improved | Worsened_or_Died | Worsened_or_Severe | Uncertain",
  "evidence": ["支持判断的原始字段值或原文短语"],
  "conflict": null,
  "reason": "一句中文判定理由",
  "confidence": "high | medium | low"
}
```

## MIMIC 记录抽取 Prompt

适用场景：输入为 MIMIC matched subset 关联到 MIMIC-III 临床表后的病例级记录，例如 `SUBJECT_ID`、`HADM_ID`、`ICUSTAY_ID`、出院去向、院内死亡字段、死亡时间、出院小结或人工整理的病程轨迹。

```text
你将收到一条 MIMIC 脑卒中相关住院记录。记录可能包含：
- SUBJECT_ID、HADM_ID、ICUSTAY_ID
- HOSPITAL_EXPIRE_FLAG、DEATHTIME、DOD
- DISCHARGE_LOCATION 或出院去向
- discharge_summary、clinical_notes 或人工整理的 trajectory_text

请抽取二分类预后 label。

MIMIC 判定规则：
1. HOSPITAL_EXPIRE_FLAG=1、存在 DEATHTIME、出院去向为 DEAD/EXPIRED，或文本明确死亡，判为 unfavorable，label_code=1，binary_prognosis="Worsened_or_Died"。
2. 文本明确 worsened、deteriorated、neurologic deterioration、condition worsened，判为 unfavorable，label_code=1，binary_prognosis="Worsened_or_Died"。
3. 出院去向为 SNF、skilled nursing facility、rehab、rehabilitation、long term care、nursing home、hospice、transfer to acute hospital，若无相反证据，判为 unfavorable，label_code=1，binary_prognosis="Worsened_or_Died"。
4. 出院去向为 HOME、HOME HEALTH CARE，或文本明确 improved/symptoms improved/discharged home，且无死亡或恶化证据，判为 favorable，label_code=0，binary_prognosis="Improved"。
5. 若只有诊断、检查、治疗经过、ICU 入出时间、波形/生命体征记录，而没有结局或出院去向，判为 uncertain。

输出合法 JSON：
{
  "record_id": "HADM_ID 或 SUBJECT_ID",
  "label": "favorable | unfavorable | uncertain",
  "label_code": 0,
  "binary_prognosis": "Improved | Worsened_or_Died | Uncertain",
  "evidence": ["原始字段或文本证据"],
  "conflict": null,
  "reason": "一句中文判定理由",
  "confidence": "high | medium | low"
}

输入：
{{mimic_record}}
```

## MC-MED 记录抽取 Prompt

适用场景：输入为 MC-MED 的病例级记录包，可由 `visits.csv`、`labs.csv`、`numerics.csv`、`rads.csv` 等按 visit 合并后提供。MC-MED 是急诊多模态数据，抽取 label 时优先使用 ED disposition、转归、出院/转院/死亡等字段或文本证据，不要从单个生命体征异常直接推断预后。

```text
你将收到一条 MC-MED 急诊脑卒中相关 visit 记录。记录可能包含：
- visit_id、patient_id
- ED disposition / disposition / outcome / transfer 字段
- diagnosis、radiology impression、notes
- labs、numerics、vitals 等早期观测值

请抽取二分类预后 label。

MC-MED 判定规则：
1. 若 disposition/outcome/text 明确 death、expired、hospice、critical deterioration、severe disability，判为 unfavorable，label_code=1，binary_prognosis="Worsened_or_Severe"。
2. 若 disposition/outcome/text 明确 transfer to acute care、ICU、inpatient rehabilitation、skilled nursing facility、long-term care、nursing home，且表示较高级别照护或严重状态，判为 unfavorable，label_code=1，binary_prognosis="Worsened_or_Severe"。
3. 若 disposition 为 home、home health、discharged home，或文本明确 improved/symptom relief/condition improved，且没有死亡、恶化或高级别照护证据，判为 favorable，label_code=0，binary_prognosis="Improved"。
4. 若 disposition 只是 admitted、observation、left without being seen、left against medical advice、unknown、missing，且没有明确预后证据，判为 uncertain。
5. labs、numerics、vitals、影像阳性结果只能作为临床背景，不能单独决定 label。

输出合法 JSON：
{
  "record_id": "visit_id 或 patient_id",
  "label": "favorable | unfavorable | uncertain",
  "label_code": 0,
  "binary_prognosis": "Improved | Worsened_or_Severe | Uncertain",
  "evidence": ["原始字段或文本证据"],
  "conflict": null,
  "reason": "一句中文判定理由",
  "confidence": "high | medium | low"
}

输入：
{{mc_med_record}}
```

## 批量抽取 Prompt

```text
你将收到一个 JSON 数组，每个元素是一条脑卒中相关记录。每条记录可能来自 MIMIC 或 MC-MED，并包含 source 字段。

任务：
请逐条抽取预后 label，返回 JSON 数组。不要遗漏记录，不要合并记录，不要输出 Markdown。

source="MIMIC" 时：
- 使用 MIMIC 规则。
- 死亡或恶化输出 binary_prognosis="Worsened_or_Died"。

source="MC_MED" 时：
- 使用 MC-MED 规则。
- 严重、恶化或高级别照护输出 binary_prognosis="Worsened_or_Severe"。

所有记录共同规则：
- favorable 对应 label_code=0，binary_prognosis="Improved"。
- unfavorable 对应 label_code=1。
- uncertain 对应 label_code=null，binary_prognosis="Uncertain"。
- evidence 必须来自输入原文或字段值。
- 不能用生命体征、实验室异常、年龄、合并症直接推断 label。

输出 JSON 数组元素 schema：
{
  "record_id": "输入记录编号",
  "source": "MIMIC | MC_MED | unknown",
  "label": "favorable | unfavorable | uncertain",
  "label_code": 0,
  "binary_prognosis": "Improved | Worsened_or_Died | Worsened_or_Severe | Uncertain",
  "evidence": ["证据1", "证据2"],
  "conflict": null,
  "reason": "一句中文判定理由",
  "confidence": "high | medium | low"
}

输入：
{{records_json}}
```

## Few-shot 示例

### MIMIC：死亡

输入：

```json
{
  "source": "MIMIC",
  "HADM_ID": "155252",
  "HOSPITAL_EXPIRE_FLAG": 1,
  "DISCHARGE_LOCATION": "DEAD/EXPIRED"
}
```

输出：

```json
{
  "record_id": "155252",
  "source": "MIMIC",
  "label": "unfavorable",
  "label_code": 1,
  "binary_prognosis": "Worsened_or_Died",
  "evidence": ["HOSPITAL_EXPIRE_FLAG=1", "DISCHARGE_LOCATION=DEAD/EXPIRED"],
  "conflict": null,
  "reason": "记录明确提示院内死亡，属于预后不良。",
  "confidence": "high"
}
```

### MC-MED：回家

输入：

```json
{
  "source": "MC_MED",
  "visit_id": "ed_001",
  "disposition": "Discharged home",
  "notes": "Symptoms improved after treatment."
}
```

输出：

```json
{
  "record_id": "ed_001",
  "source": "MC_MED",
  "label": "favorable",
  "label_code": 0,
  "binary_prognosis": "Improved",
  "evidence": ["disposition=Discharged home", "Symptoms improved after treatment"],
  "conflict": null,
  "reason": "记录提示治疗后改善并出院回家，属于预后良好。",
  "confidence": "high"
}
```

### 证据不足

输入：

```json
{
  "source": "MC_MED",
  "visit_id": "ed_002",
  "diagnosis": "ischemic stroke",
  "vitals": {"SBP": 180, "SpO2": 93}
}
```

输出：

```json
{
  "record_id": "ed_002",
  "source": "MC_MED",
  "label": "uncertain",
  "label_code": null,
  "binary_prognosis": "Uncertain",
  "evidence": [],
  "conflict": null,
  "reason": "输入只有诊断和生命体征，缺少明确结局或出院去向证据。",
  "confidence": "low"
}
```

## 质控清单

- 抽取后检查 `LABEL=0` 是否均对应 `Improved`。
- 抽取后检查 `LABEL=1` 是否对应 `Worsened_or_Died` 或 `Worsened_or_Severe`。
- 对 `uncertain`、低置信度、证据冲突样本进行人工复核。
- 建模前剔除无法明确二分类的样本。
